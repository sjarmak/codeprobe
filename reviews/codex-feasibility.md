**Key Rework Needed**

The PRD should tighten these before implementation:

1. Define the exact metadata source of truth. Today `execute_task()` reads only `metadata.json`, while R3 specifically mentions `task.toml`.
2. Define `DualScorer` configuration: binary direct leg by default, optional continuous direct leg, and where that setting lives.
3. Decide the serialized shape of typed `DualScoringDetails`. A frozen dataclass is useful in Python, but checkpoint/report JSON needs a stable dict schema.
4. Reconcile NG5 with R11. If artifact scores must appear in text/HTML/CSV/JSON reports, the analysis/report layer is in scope.
5. Standardize ground truth location for dual tasks as `tests/ground_truth.json`; otherwise writer, validator, and artifact scorer will keep accepting different layouts.
